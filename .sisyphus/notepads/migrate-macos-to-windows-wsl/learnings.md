# 重构学习笔记：WSL 迁移指南从教程到清单

## 重构原则

### 1. 用户体验优先
- 用户自己写的体验、感受、软件推荐 = **完全保留**
- 技术配置内容 = **简化为清单**
- 这种区分基于内容的**重用价值**和**个人特色**

### 2. 代码块删除策略
- 删除大块的配置文件完整内容（例如完整的 ini 文件配置）
- 删除冗长的 PowerShell 命令序列
- **保留**关键命令的行内代码形式：`wsl --install`
- **保留**必要的配置代码块（如实际使用工作流、.zshrc 关键配置）

### 3. 清单化技巧
- 使用编号列表表示步骤流程
- 使用 `-` 表示要点或特点
- 使用 `**粗体**` 强调关键概念
- 使用行内代码 `` `code` `` 展示命令

## 保留标准

### 必须完整保留的内容
1. **用户的主观体验**（不可复制，高价值）
   - Windows 开发体验心得
   - 常用软件推荐理由
   - 迁移后的感受总结

2. **个人特色配置**（展现个人风格）
   - 使用 Clash for Windows 的代理方案
   - Zsh 的插件选择和配置
   - VSCode 扩展的精选

### 可以简化的内容
1. **通用步骤**（可通过搜索获得）
   - 安装命令可以改为列表
   - 配置文件格式可以改为说明

2. **代码块**（不是核心价值）
   - 冗长的安装脚本
   - 完整的配置文件
