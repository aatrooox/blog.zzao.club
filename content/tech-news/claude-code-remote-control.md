我刚看到 Claude Code 上线了一个新功能：**Remote Control**。

简单说：
你让 Claude 在你电脑上干活。
你用手机当遥控器。

就这么朴素。
但挺爽。

## 这功能到底是啥

Remote Control 不是“在手机上跑 Claude”。
也不是把你的代码丢到云上。

它的思路更像：
- Claude Code **一直在你本机终端里跑**（本地仓库、本地权限、本地环境都在）
- 你在手机（Claude App 或网页）上接管“交互/确认/继续下指令”

人离开电脑了。
任务不断线。

## 怎么用（最短路径）

我看到的用法大概是：

1) 在终端里启动 Remote Control：`claude rc`
2) 终端会生成一个链接或二维码
3) 你用手机扫一下/打开链接
4) 然后就能从手机继续控制这次会话

入口页面也给了：
https://claude.ai/code

## 安全性：不是“谁拿到 URL 谁就能控你电脑”

这点很关键。

博主的解释是：两边需要登录**同一个 Claude 账号**。
也就是说，就算别人知道你的 URL，也不代表他就能接管你的会话。

（当然，任何“远程控制”都值得你多一层警惕：用完就关，别长期开着。）

## 目前谁能用

现阶段据说是 **Max 订阅用户**先用，Pro 还要等等。
以官方后续说明为准。

## 我觉得最有用的两个场景

### 1) 走动/开会间隙：把等待变成推进

跑构建、跑测试、装依赖这种事，你懂。

以前你离开电脑就等于断线。
现在你可以边走边看它跑到哪了，卡住就补一句继续。

这才是重点。

### 2) 临门一脚：不用回工位也能收尾

有些任务最后就差你一句“继续/确认/把 A 改成 B”。

Remote Control 让你不用坐回键盘前，也能把事情结掉。

## 一个提醒

Remote Control 的前提是：Claude Code 在你本机拥有什么权限，你远控时就等于在使用同样的权限。

所以我个人会坚持三条：
- 尽量别在公共 Wi‑Fi 上开
- 用完就关
- 涉及密钥/删除/生产环境操作，最好回到电脑前二次确认

## 我的结论

这不是更强的模型。
但它是更像“工具”的一步。

值。
