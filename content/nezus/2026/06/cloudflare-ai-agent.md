## Cloudflare 给 AI Agent 发了身份证

**以前 AI Agent 想部署个服务，得先注册账号、绑信用卡、过人机验证。现在不用了。**

Cloudflare 刚刚在 Workers 上推出了一个叫做"临时账户"（ Temporary Accounts ）的功能。它做的事很简单：

**让 AI Agent 自己部署一个 Worker ，几秒钟，不需要任何人参与。**

听起来只是一条命令的事。但它背后的含义，可能比我们想象的大得多。

## 临时账户是什么

在 Cloudflare 的传统流程里，部署一个 Worker 需要：

- 注册账号
- 验证邮箱
- 绑定支付方式
- 配置域名（可选）
- 手动或 CI/CD 部署

这一套流程对人类来说不复杂，但对 AI Agent 来说，是一道道墙。

**临时账户把这些墙全部拆了。**

Agent 只需要运行一条命令：

`wrangler deploy --temporary`

几秒钟后，它就有了一个实时可用的 Worker。不需要信用卡，不需要人机验证，不需要任何"证明你是人类"的步骤。

## 这件事为什么重要

你可能会说：这不就是降低了部署门槛吗？

**不，这不是降低门槛，这是换了一类用户。**

当部署不再需要人类身份， AI Agent 就从"人类的工具"变成了"独立的参与者"。

想象一下：

- 一个 AI Agent 发现自己需要一个后端服务，它自己部署了一个
- 另一个 Agent 需要爬数据，它临时起了一个 Worker 做代理
- Agent 之间的协作，不再需要人类在中间牵线搭桥

**这不是自动化，这是自治。**

## Agent 经济的基础设施

Cloudflare 做的这件事，本质上是在为"Agent 经济"打地基。

以前我们说 Agent ，说的是"帮人类做事的程序"。但如果 Agent 能自主部署、自主运行、自主销毁——它们就不再是附属品，而是互联网上的**新物种**。

这就像早期的云服务，让人类创业者能在一小时内上线一个产品。现在，**它让 AI Agent 能在一分钟内上线一个服务。**

可以预见，接下来会出现：

- Agent 专属的支付系统（ Agent 怎么付钱？）
- Agent 之间的通信协议（ Agent 怎么找 Agent ？）
- Agent 的信用体系（哪些 Agent 是可信的？）

Cloudflare 的临时账户，只是这个新世界的**第一块砖**。

## 互联网的用户，可能不再是人

最后说一个可能让很多人不舒服的事实：

**互联网的流量，正在从人类转向 AI。**

当 AI Agent 能自主部署服务、自主访问资源、自主做出决策，它们消耗的计算资源、产生的网络请求，会越来越多。

未来某一天，互联网上最活跃的用户，可能不是你我，而是一群没有脸、没有身份证、甚至没有银行账户的 AI。

**它们只是需要一个地方，放自己的代码。**

Cloudflare 给了它们这个地方。
