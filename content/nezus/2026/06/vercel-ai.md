## Vercel 悄悄扔了个炸弹：一个文件夹，就是一个 AI 员工

上周五晚上，我刷 Twitter 的时候差点错过这条消息。

Vercel——就是那个让全世界前端开发者离不开的部署平台——悄悄开源了一个叫 Eve 的东西。没有发布会，没有预热海报，就是一个 npm 包， Apache-2.0 协议，随便你用。

但当我花了两小时读完文档之后，后背有点发凉。

因为这玩意，可能重新定义"怎么造一个 AI 员工"。

## 一个文件夹=一个智能体，就这么简单

传统的智能体框架怎么搞？你要写配置文件、注册工具、声明能力、连接 API、写一堆胶水代码。

Eve 说：不用了。

它的核心设计叫"文件系统优先"。你想造一个智能体？在磁盘上建一个文件夹就行。文件夹里放什么，智能体就会什么。模型、指令、工具、技能、外部连接、甚至子智能体——全是文件和目录。

我第一次看到的时候以为在开玩笑。但仔细一想，这不就是 Unix 哲学吗？一切皆文件。

没有框架锁死，没有 DSL 要学，没有 YAML 地狱。你 ls 一眼就知道这个智能体能干嘛。你用 vim 就能改它的行为。你 cp 一个文件夹就能克隆一个智能体。

这不是简化，这是降维。

## 六个能力，全是"生产级"的

如果只是"文件夹映射"这种花活，那 Eve 也就是个玩具。但 Vercel 在里面塞了六个硬核能力，每一个都是踩过大坑才知道要做的东西：

**持久执行。** 每一步自动检查点，进程挂了？重启后从上一步继续。不是从头跑，是真的从断点恢复。你知道在生产环境里，一个跑 4 小时的智能体任务崩溃后要从头来的绝望吗？

**沙箱计算。** 智能体要跑代码？行，给你隔离环境。不会把你的数据库炸了，不会把你的服务器搞崩。

**人机审批。** 关键动作可以设置"暂停等人批准"。AI 做决策，人来把关。这才是企业真正敢上线的东西。

**安全连接。** 原生支持 MCP 和 OpenAPI 协议。不用自己写 adapter ，不用自己搞鉴权。

**多通道。** Slack、Discord、Teams……一个智能体可以同时活在多个渠道里。

**追踪与评估。** 基于 OpenTelemetry ，每一步操作都有 trace。出了问题你能回溯，能分析，能优化。

说实话，这六条单拿出来任何一个，都够某些创业公司做一个产品了。Vercel 把它们打包塞进了一个 npm 包。

## 他们自己先用了，而且用得很狠

Vercel 不是在放 PPT。他们内部已经跑了上百个智能体，有几个数字相当炸裂：

**d0** —— 一个数据分析智能体，每月处理超过 3 万次查询。注意，这不是实验环境，是真实业务数据。

**Lead Agent** —— 自动销售代理。年费大概 5000 美元（主要是 API 调用费），但它带来的销售回报是成本的 32 倍。你算算， 16 万美元的回报。

**Vertex** —— 客户支持智能体，自主解决率超过 90%。也就是说， 10 个客户问题里 9 个以上它自己就搞定了，根本不需要人介入。

这些数字背后有一个残酷的事实： Vercel 已经用 AI 智能体替代了大量人力，而且效果远超预期。

## 为什么是现在？

你可能会问：智能体框架又不新鲜， LangChain、CrewAI、AutoGen……一抓一大把， Eve 凭什么不一样？

我的观察是：之前的框架都在解决"怎么让 AI 更聪明"的问题，而 Eve 在解决"怎么让 AI 更可靠"的问题。

聪明不值钱。GPT-4 够聪明了， Claude 够聪明了。但你把一个聪明的 AI 扔到生产环境里，它会崩溃、会卡死、会干出离谱的事、会花光你的预算。

Eve 的每一个设计——检查点、沙箱、审批、追踪——都是在回答同一个问题：怎么让老板敢把智能体上线？

这可能是开源社区第一次认真回答这个问题。

## 对普通开发者意味着什么

如果你是个全栈开发者， Eve 对你的意义可能比你想象的大。

以前你要给公司做一个"智能客服"或者"数据分析助手"，你得搭一套复杂的系统。现在？建个文件夹，写好指令，挂上工具，部署到 Vercel （或者你自己的服务器），完事。

Apache-2.0 意味着你可以随便改、随便用、商用也行。

我觉得接下来几周， GitHub 上会涌现大量基于 Eve 的"智能体模板"。就像当年 WordPress 模板生态一样，只不过这次不是网站，是 AI 员工。

## 写在最后

我不是说 Eve 会一统天下。智能体领域还太年轻，格局远未确定。

但 Vercel 这一手确实漂亮。他们没有去做"最聪明"的框架，而是做了"最能落地"的框架。在 AI 越来越强的今天，"能落地"比"能跑分"值钱一万倍。

如果你在做 AI 产品，或者你们公司正在考虑引入智能体，我建议你去翻翻 Eve 的源码。不用全看懂，感受一下那个"文件夹即智能体"的设计哲学就够了。

链接在这： npm 搜 eve ， GitHub 搜 vercel/eve。

反正免费，看看又不亏。
