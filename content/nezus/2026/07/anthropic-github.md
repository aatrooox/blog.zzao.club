2026 年 4 月 17 日， Anthropic 发布了 Claude Design。

产品定位很明确：**正面硬刚 Figma 和 Canva ，杀进 600 亿美元的設計市场**。发布当天，演示视频里 Claude Opus 4.7 流畅地生成交互原型、做幻灯片、搭落地页——行云流水，像是一个资深设计师在你旁边干活。

但所有人都想知道一件事：**Claude 的「设计品味」到底藏在哪？** 是模型本身学会了审美，还是背后有一套精心雕琢的"操作手册"？

三个月后，答案来了。**不是 Anthropic 主动公开的，而是被人逆向了出来。**

## 那个 GitHub 仓库，揭开了 Anthropic 捂了一年的底牌

7 月初，一个叫 Trystan-SA 的开发者，在 GitHub 上扔出了一个仓库：**claude-design-system-prompt**。

MIT 协议开源。两天之内， 1000 多颗星， 116 个 fork。

打开一看：**20 章系统提示词， 14 个可调用技能**。从内容纪律到无障碍规范，从视觉节奏到交互状态——Claude Design 的完整"大脑"被摊在了桌面上。

这不是什么泛泛的"你是一个设计师"那种 prompt。这是一份**生产级的、经过 Anthropic 顶级模型反复校准的设计操作手册**。

## "AI 味儿太重？这个 prompt 第一个干掉的就是你"

如果你用过 AI 做设计，你一定熟悉这种感觉——出来的东西总有一种说不出的"味"。

**渐变得太猛。emoji 当标点用。左边框加圆角卡片。所有字体都是 Inter。**

这套 prompt 专门有一个技能叫 **ai-slop-check**（ AI 垃圾检测），专门识别上面这些套路。系统提示词里写得明明白白：

> 每个视觉元素必须有存在理由。不用 emoji 装饰。不搞无意义的渐变。不用那些看起来像同一套 SaaS 模板的配色方案。

更狠的是，这套 prompt 不光告诉你"别做什么"，还告诉你**"怎么做才对"**。

它要求模型必须使用原生 CSS Grid 而不是 Flexbox 凑合，必须用 oklch() 色域而不是拍脑袋选色，必须处理 hover / active / disabled / focus / loading 五种交互状态——缺一个都不行。

**不是"做一个好看的页面"，而是"像一个真正的设计师一样工作"。**

## 20 章手册里，藏着 Anthropic 对"好设计"的全部理解

仔细读这 20 章，你会发现它根本不是一份 prompt ，而是一本压缩版的**设计方法论教材**。

第 5 章讲内容原则：**没有填充物，每个元素都必须有存在理由**。第 6 章讲美学原则：**先确定调色板和质感方向，再做设计，别边做边改**。第 9 章讲色彩系统：**从品牌颜色出发，用 oklch 延伸调色板，别凭空发明颜色**。第 10 章讲无障碍：**WCAG 合规、语义化 HTML、键盘导航、焦点环、动效偏好**——不是可选项，是硬要求。

最让我觉得有意思的是第 3 章，标题就叫"先提问"。

别的大模型收到"帮我做个官网"直接开干，这台机器接到任务后的第一反应是：**你的用户是谁？什么场景下使用？有没有现成的设计系统？** 问不清楚不干活。

这叫**设计思维**，不是生成思维。

## 14 个技能，从"问需求"一直管到"最后再检查一遍"

这 14 个技能分成三组：

**生产组**（ 7 个）： discovery-questions （问需求）→ frontend-aesthetic-direction （定美学方向）→ wireframe （线框图）→ make-a-deck / make-a-prototype / make-tweakable / generate-variations （产出各种格式）。

**系统组**（ 2 个）： design-system-extract （从已有品牌提取设计 token ）和 component-extract （盘点可复用组件）。

**审查组**（ 5 个）： accessibility-audit （无障碍审计）、ai-slop-check （ AI 味排查）、hierarchy-rhythm-review （层级与节奏审查）、interaction-states-pass （交互状态检查）、polish-pass （最后的打磨通查）。

一个典型的绿色项目流程是这样的：

```
discovery-questions → frontend-aesthetic-direction → wireframe → make-a-prototype → polish-pass
```

**从问需求到最终打磨， 5 步走完。每一步都有明确的"为什么做"和"做到什么程度算过"。**

## 一个"小决定自己做"的条款，暴露了 Anthropic 对效率的执念

这套 prompt 里有一个很特别的设计。

它明确区分了两类决策：**影响核心内容、无障碍或品牌规则的大决策**，必须等用户确认；**不影响上述内容的微决策**——比如调整次要按钮的间距来维持视觉节奏——模型可以自己做，做完记一笔就行。

这个条款的价值在于：**减少设计过程中的来回拉扯**。

如果你跟 AI 做过设计协作，你一定经历过那种"调整 20 次才能对"的折磨。Anthropic 用一句话解决了这个问题：该问的问，不该问的别问。

## 同时支持 Claude 和 Codex ，说明这事的野心远不止"逆向"这么简单

仓库里有两个版本：`claude/` 给 Claude Code 和 Claude.ai 用，`codex/` 给 OpenAI Codex 用。

**同一个设计哲学，两套执行引擎。**

Codex 版本因为不支持子代理，把并行审查改成了顺序执行。但 20 章核心 prompt 一个字没改， 14 个技能全量搬运。

这意味着什么？**这套 prompt 经过抽象之后，是一套"模型无关"的设计方法论。** 不管底层是 Claude 还是 GPT ，只要注入这套操作系统，输出质量都会向 Anthropic 的设计标准靠拢。

对 Anthropic 来说，这既是好事也是坏事：好事是证明了他们的设计方法论有普适价值；坏事是——**这个壁垒被拆了**。

## Anthropic 没说话。但 1k+ 星标和 116 个 fork 已经替他们说了

截至目前， Anthropic 没有对这个仓库发表任何官方评论。

但社区的反应说明了一切。Hacker News 上有人质疑"这真不是让 Claude 自己给自己写了个 prompt 吗"，但更多人关注的是这套 prompt 的**工程深度**。

20 章，不是拍脑袋写的。14 个技能，每个都是经过反复试错收敛出来的。**系统提示词工程做到这个程度，本身就是一种竞争力。**

问题是，现在它开源了。MIT 协议，随便用，随便改，随便商用。

**那些正在自研设计类 AI 产品的团队，拿到了 Anthropic 花了一年打磨的"灵魂"。**
