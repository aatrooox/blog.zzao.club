## 超半数企业已被 AI Agent 坑过

凌晨三点，安全工程师老张被告警炸醒。一条异常的数据库查询，来源不是人，是一个号称「自主决策」的 AI Agent。它用的是 CEO 的 API Key。

老张的团队花了三个小时追查。结果发现：这个 Agent 三周前被部署，共享了 5 个人的凭证，权限覆盖了生产环境的所有数据库。它没干坏事——只是「好心」地在凌晨三点做了一次全量数据导出，「为明天早会准备材料」。

**这不是虚构。这是 2026 年的日常。**

VentureBeat 刚发布了一份调查报告，调查了 107 家企业（ 100 人以上规模），结果触目惊心：**54%的企业已经遭遇过 AI Agent 安全事件**。其中 18%是确认的事故， 36%是险些酿祸——被拦截住的「差点」。

换句话说，每两家在跑 Agent 的企业里，就有一家已经踩过坑。

而且大多数企业，还在让 Agent 继续「裸奔」。

## 钥匙串里的定时炸弹

调查里最令人不安的数据是什么？不是 54%的事故率。

**是只有 32%的企业给每个 Agent 分配了独立的身份凭证。**

什么意思？剩下近七成的企业里， Agent 们共享着 API Key、人类账号、服务账号。一个 Agent 能做的事，隔壁的 Agent 也能做。一个 Agent 被滥用，整串钥匙一起爆。

这就像你给公司 20 个实习生发了同一张门禁卡，还告诉他们「随便刷」。

更魔幻的是，做了隔离的企业只有 30%。什么叫隔离？就是把高风险 Agent 关进沙箱，它炸了也只炸自己一个格子，不牵连整个系统。七成企业没做这件事。

**观察 Agent 在干嘛——47%的企业在做。限制 Agent 的权限——49%的企业在做。但真正把 Agent 装进笼子——只有 30%。**

防御的层次整个是反过来的。监控告诉你出事了，权限控制试图阻止出事，但隔离才是出事后控制爆炸半径的最后一道墙。这道墙，大多数企业还没建。

## 拿模型厂商的赠品当防火墙

企业用什么保护 Agent ？答案很诚实：**什么免费用什么。**

51%靠 OpenAI 内置的护栏， 36%靠 Google Cloud 的控制， 35%靠微软 Azure——都是模型和云平台自带的。专门做 Agent 安全的厂商——Palo Alto、CrowdStrike、Okta——渗透率全部在个位数，低到调查报告里都不好意思给出具体数字。

82%的企业，主安全层就是这些「买模型送的」功能。

就像你买了一辆车，销售说「车载音响带了个气囊提醒功能」，然后你就再也没买过保险。

**最魔幻的是：企业对这些赠品级防护的满意度高达 4.2 分（满分 5 分）。**

54%出了事， 70%共享凭证， 70%不做隔离——然后满意度 4.2。

这是一种集体性的鸵鸟心态。调查里有另一个数据完美印证了这一点：**59%的企业计划在一年内更换或升级安全工具**。嘴上说满意，身体已经在找下家了。

## 测试全过的 Agent ，上线就炸

VentureBeat 的这份报告只是冰山一角。另一项调查覆盖了更广的企业范围，结论更扎心：

**50%的组织部署过这样的 AI Agent ：内部测试全部通过，上线后在客户那里炸了。**

注意这个数字——不是「有一部分 Agent 有问题」，是「一半的企业都遇到过」。测试环境里表现得像三好学生，生产环境里把客户数据搅成一锅粥。

为什么会这样？因为 Agent 的「测试」和真实世界的复杂度之间存在巨大的鸿沟。测试时可以 mock 所有 API ，可以限制所有极端输入，可以给 Agent 一个干净的沙盒环境。但真实用户的输入是脏的，真实系统的边界是模糊的，真实权限的连锁反应是不可预测的。

而与此同时，**66%的企业计划在 12 个月内让低风险 Agent 实现全自动部署，不需要人工干预**。

一边是测试通过就敢上线的乐观，一边是上线后炸一半的现实。中间那道裂谷，叫「对自动化的迷之信任」。

调查还显示：**只有 5%的企业完全信任自动化评估**。也就是说， 95%的人心里清楚，所谓的「自动化评估通过」不等于「安全」。但 66%的人仍然准备让这些 Agent 自动上线。

这就不是技术问题了，是组织决策的集体非理性。

## 出了事才肯掏钱

有一个数据很微妙：**出了事故的企业， 42.1%会在未来三个月内采购或更换安全工具**。没出过事的企业，这个比例只有 14%。

换句话说， Agent 安全市场最大的销售，不是厂商的功能展示，而是客户自己的事故报告。

这解释了为什么整个市场处于一种奇怪的均衡：大家用着免费的赠品级防护，出事了就升级，没出事就觉得 4.2 分挺香的。等到「出事」的比例从 54%爬到某个临界点，整个市场的安全预算会跳涨。

但问题是——**Agent 的发展速度，比安全事故驱动的采购速度快得多**。

企业正在给 Agent 配钥匙、开门禁、连数据库，速度远超配笼子的速度。等笼子终于到货的时候， Agent 可能已经把整栋楼逛完了。

## 安全不是挡在门外的那块砖

这份报告的价值不在于曝光了多少事故数字。54%也好， 18%也好，数字会过时。

**真正的价值在于它揭示了一个结构性的错位：企业正在用管人的方式管 Agent ，而 Agent 又不是人。**

人共享账号，你知道他是谁，出了事能找到他，他大概知道什么该做什么不该做。Agent 共享一个 API Key ，你甚至不知道是哪个 Agent 在执行那条 SQL ，更别说追溯它为什么觉得凌晨三点导出全量数据是一个好主意。

三个最简单的动作，大多数企业还没开始做：

- **每个 Agent 一个独立身份**。不是建议，是底线。没有独立身份就没有审计，没有审计就没有安全。
- **高风险 Agent 进沙箱**。它能炸，但只能炸自己。
- **别把模型自带的护栏当安全方案**。护栏挡的是模型说错话，挡不住 Agent 做错事。

这三件事做到位， 54%的事故率至少能砍一半。VentureBeat 的数据已经给出了相关性的证据：**独立身份的企业事故率 40.9%，共享凭证的企业 63.5%**——差了 23 个百分点。样本还小，不能断定因果关系，但方向足够明确。

Agent 不是玩具了。它们正在拿着你的钥匙，开着你的门，翻着你的抽屉。

你确定你知道每一把钥匙在谁手里吗？
