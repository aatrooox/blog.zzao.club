**上个月，一个程序员在 GitHub 上发了个帖子，说他用 Claude 自动发了条推特。评论区炸了。**

不是因为技术有多难。是因为 X （前 Twitter ）官方终于松口了。

## 发生了什么

X 正式开放了 MCP （ Model Context Protocol ）接口。说人话就是： AI 智能体现在可以直连推特了。

不是爬虫，不是模拟器，不是第三方桥接。是 X 官方给的通道。

读推文、发推文、回复、私信、搜索——AI 全都能做，而且用的是正经的 API 权限。

这意味着什么？

你让 Claude 帮你盯着某个 KOL 的最新发言，自动生成摘要。可以。你让 GPT 每天定时发一条 AI 行业简报。可以。你让一个 Agent 自动回复客户私信， 7×24 小时不睡觉。也没问题。

## 但这事没那么简单

X 开放 MCP 这个动作，不是突然发善心。

2023 年马斯克关闭了免费 API ，逼得无数开发者和研究者骂娘。2024 年 API 定价高到离谱，小团队根本用不起。2025 年 X 的用户增长放缓，广告收入承压。

然后到了 2026 年， AI Agent 成了全行业的关键词。OpenAI、Anthropic、Google 都在推 Agent 生态。如果 X 再不开门， AI 就会绕过它——用户通过浏览器插件、自动化脚本、第三方工具让 AI 操作社交网络， X 一分钱都收不到。

所以这次开放 MCP ，本质上是一次"主动纳管"。

与其让 AI 在围墙外面乱窜，不如开门收门票。

## 有意思的地方在哪里

MCP 是 Anthropic 提出的开放协议，本来是为 Claude 设计的。但现在 X 用的是同一个协议标准。

也就是说，任何支持 MCP 的 AI——Claude、GPT、开源模型——都能用同一套接口连上 X。

这对 Anthropic 是个利好（ MCP 成了事实标准），但对 OpenAI 未必是坏事（生态大了大家都受益）。

真正被动的可能是 Meta。

Threads 至今没有对 AI Agent 开放任何正经接口。如果 X 的 MCP 生态跑起来， Threads 的反应窗口可能只有几个月。

## 真正的隐忧

一个 AI Agent 能发推、回复、私信、搜索——听起来很方便。但你想想：

如果一个 Agent 被用来批量注册账号、自动生成争议性内容、精准回复特定话题来操控舆论方向——谁来管？

X 说会有速率限制和内容审核。但说实话，以 X 目前的审核团队规模（裁了好几轮），我不太相信它能兜住底。

所以结论很简单：**工具是中性的，但使用工具的人从来不是。**

MCP 开放是好事。它让正经开发者和企业能用上 AI 的社交能力。但也一定会有人把它用歪。

2026 年下半年，我们大概率会看到第一波"MCP + X"的滥用案例。到时候再看 X 怎么应对。
